# Catch dafr up to DataAxesFormats 0.3.0

Date: 2026-07-05

## Problem

dafr (the JuliaCall wrapper) was written against DataAxesFormats.jl v0.1.2/v0.2.0.
The Julia package is now v0.3.0 and the runtime (Julia 1.12, also used by CI)
dev-tracks the local 0.3.0 checkout, so the new features are available - the gap
is purely missing R wrappers. v0.3.0 added chunked+compressed "packed" storage,
and Zarr / ZIP / HTTP backends. v0.2.0 added axis reordering and contract-
enforcement control that were never wrapped.

## Scope

In:

- Storage backends: `zarr_daf`, `zip_daf`, `http_daf`.
- `packed` support threaded into `files_daf`, `h5df`, the new backends, `open_daf`,
  and `complete_daf`.
- `open_daf` dispatch fix - delegate to Julia's `open_daf` instead of the current
  stale R reimplementation.
- Format conversion: `files_to_zarr`, `zarr_to_files`.
- Axis reordering: `reorder_axes`, `reset_reorder_axes`.
- Config: packing knobs + `DAF_ENFORCE_CONTRACTS`.

Out (deliberately):

- Merge actions (`SkipProperty`/`LastValue`/`CollectAxis`) - `concatenate` already
  exposes these as strings.
- `AllOf`/`RangeOf`/`IndicesOf`/`MaskOf` (`SparseSelection`) and `LazySparse*` -
  internal plumbing for the lazy-sparse path, no user-facing R call.
- Flame-graph / performance tracking - Julia-side dev tooling.

## Components

### R/formats.R (edit)

Three thin constructors mirroring `files_daf`, plus `packed = FALSE` threaded into
`files_daf` and `h5df`:

- `zarr_daf(path, mode = "r", name = NULL, packed = FALSE)` -> `DataAxesFormats.ZarrDaf`
- `zip_daf(path, mode = "r", name = NULL, packed = FALSE)` -> `ZipDaf`
- `http_daf(url, name = NULL, packed = FALSE)` -> `HttpDaf`. Read-only; no `mode`
  argument, matching the Julia signature.
- `files_to_zarr(files_path, zarr_path)`, `zarr_to_files(zarr_path, files_path)` -
  keyword-arg calls to the Julia functions. Paths normalized with `normalizePath`.

`zarr_daf`/`zip_daf` normalize local paths but must NOT normalize `http(s)://`
Zarr URLs (ZarrDaf accepts remote paths). `http_daf` leaves the URL untouched.

### R/complete.R (edit)

- `open_daf(path, mode = "r", name = NULL, packed = FALSE)`: delete the R-side
  if/else dispatch. Normalize local paths (skip normalization for `http(s)://`),
  then delegate to `DataAxesFormats.open_daf(path, mode; name, packed)`.
- Verify a `mode = "w"` files round-trip works under 0.3.0. The existing
  `files_daf` unlink workaround was for a v0.2.0 truncate/cache bug; Julia's
  `parse_mode` handles `"w"` truncation natively. Keep a pre-unlink guard in
  `open_daf` only if the round-trip test shows the bug survives.
- `complete_daf(leaf, mode = "r", name = NULL, packed = FALSE)`: add `packed`,
  forward to `DataAxesFormats.complete_daf`.

### R/reorder.R (new)

- `reorder_axes(daf, permutations)`: `permutations` is a named list mapping axis
  name -> integer permutation vector (1-based, same as Julia). Accepts a single
  Daf or a list of Daf. Builds a `Dict{String, Vector{Int}}` on the Julia side and
  calls `DataAxesFormats.reorder_axes!`.
- `reset_reorder_axes(daf)`: calls `DataAxesFormats.reset_reorder_axes!`; returns
  the logical it returns (whether a stale reorder was rolled back).

### R/config.R (new)

Two get/set functions rather than six single-value exports:

- `daf_packed_options(compression = NULL, compression_level = NULL,
  target_chunk_kb = NULL, local_cache_kb = NULL, http_cache_kb = NULL,
  http_max_coalesce_gap_kb = NULL)`: `NULL` args leave the corresponding
  `DataAxesFormats.PackedFormat.DAF_PACKED_*` / `DAF_HTTP_MAX_COALESCE_GAP_KB`
  global unchanged; assigns the others; returns the current values invisibly as a
  named list. `compression` is a Julia `Symbol` (e.g. `"blosc_zstd_bitshuffle"`,
  `"gzip_shuffle"`) passed through as a string and converted to `Symbol` in Julia.
- `enforce_contracts(enforce = NULL)`: get/set
  `DataAxesFormats.Contracts.DAF_ENFORCE_CONTRACTS`; `NULL` returns current value.

### Docs / packaging

- roxygen `@export` on all new functions; regenerate `NAMESPACE`.
- `NEWS.md` entry.
- Bump the `v0.2.0` doc links touched in edited files to `v0.3.0`.

## Testing (testthat)

- Zarr write -> read -> verify round-trip (scalar + axis + vector + matrix).
- Zip write -> read -> verify round-trip.
- One `packed = TRUE` round-trip (files backend) reads back identical data.
- `open_daf` picks the right backend per suffix (`.daf.zarr`, `.daf.zip`, `.h5df`,
  plain dir) - assert the returned object round-trips, not the class name.
- `reorder_axes` permutes a small daf; entries follow the permutation.
- `reset_reorder_axes` returns `FALSE` on a clean daf (nothing to reset).
- Conversion files -> zarr -> files preserves data.
- Config setters read back what they set; `NULL` leaves values unchanged.
- HTTP: `http_daf` on a non-`http` path errors; `open_daf(url, mode = "r+")`
  errors. Live fetch test guarded by `skip_on_cran()` + `skip_if_offline()` (needs
  a served fixture R CI won't have).
